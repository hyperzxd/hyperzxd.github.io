Hadoop+Spark配置

  1. 用户组
    groupadd hadoop
    useradd hadoop -g hadoop
    注意: 为新添加用户设置密码 passwd hadoop

  2. jdk安装
    tar xvf jdk-xxx.tar.gz
    cp －R jdk-xxx /opt/java／
    注意：/opt/java目录下面要有jdk这个文件夹, 目录结构不要搞错

添加环境变量
vi /etc/profile 添加
export JAVA_HOME=/opt/java/jdk
exprot PATH=$JAVA_HOME/bin:$PATH
注意：改变profile后，每次切换到这个用户要执行
source /etc/profile
如果想每次自动加载环境变量，就修改 ~/.bashrc 添加
sourec /etc/profile

修改java目录的所有者
chown -R hadoop:hadoop /opt/java
注意：要用－R选项，使下面所有文件夹都改过来。易出错。

  1. hadoop安装
    tar xvf hadoop-xxx.tar.gz
    cp －R hadoop-xxx /opt/hadoop

添加环境变量
vi /etc/profile 添加
export HADOOP_HOME=/opt/hadoop
export PATH=$HADOOP_HOME/bin:$PATH
注意：改变profile后，每次切换到这个用户要执行
source /etc/profile
如果想每次自动加载环境变量，就修改 ~/.bashrc 添加
sourec /etc/profile

修改hadoop目录的所有者
chown -R hadoop:hadoop /opt/hadoop
注意：要用－R选项，使下面所有文件夹都改过来。易出错。

  1. 修改hadoop配置文件
    su hadoop， 一定要切换到hadoop用户处理下面操作，不然会污染文件权限。别忘了source /etc/profile

vi /opt/hadoop/etc/hadoop/hadoop-env.sh
export JAVA_HOME=/opt/java/jdk

vi /opt/hadoop/etc/hadoop/core-site.xml


hadoop.tmp.dir
/hadoop


fs.default.name
hdfs://master:9000


dfs.name.dir
/hadoop/name

注意加到xml文件中间, 还有注意端口使9000在这里配置。

vi /opt/hadoop/etc/hadoop/hdfs-site.xml

dfs.replication
3


dfs.data.dir
/hadoop/data

vi /opt/hadoop/etc/hadoop/mapred-site.xml


mapred.job.tracker
master:9001


mapred.system.dir
/hadoop/mapred_system


mapred.local.dir
/hadoop/mapred_local

vi /opt/hadoop/etc/hadoop/masters
master

vi /opt/hadoop/etc/hadoop/slaves
master
slaves1
slaves2

  1. 配置ssh无密码连接

修改ip对应主机名
vi /etc/hosts
192.168.1.11 master
192.168.1.12 slave1
192.168.1.13 slave2

切换hadoop用户（重要），别忘了source /etc/profile
su hadoop; source /etc/profile
mkdir -p /user/hadoop/.ssh 建立多级文件夹用-p选项
cd /user/hadoop/
生成密钥
ssh-keygen -t rsa -P ‘’ -f ~/.ssh/id_rsa
cp id_rsa.pub authorized_keys (重要）
拷贝到其他机器
scp authorized_keys hadoop@slave1:/home/hadoop/.ssh
scp authorized_keys hadoop@slave2:/home/hadoop/.ssh

注意：所有机器的.ssh目录的所有者(hadoop)和权限一定要正确

测试：
ssh slave1
ssh slave2

可能需要关闭防火墙
1) 重启后生效
开启： chkconfig iptables on
关闭： chkconfig iptables off

2) 即时生效，重启后失效
开启： service iptables start
关闭： service iptables stop

  1. 运行hadoop
    切换用户，source

格式化
cd /opt/hadoop/bin
./hadoop namenode -format

运行
cd /opt/hadoop/sbin
./start-all.sh

测试
master 机器 jps

SecondaryNameNode
JobTracker
TaskTracker
NameNode
DataNode
Jps

slave 机器 jps
DataNode
Jps
TaskTracker

注意：执行jps结果错误，可能使用户没有切换和没有source

访问http://master:50070可以查看分布式文件系统的状态

运行WordCount
cd /opt/hadoop
hadoop fs -mkdir -p input
hadoop fs -copyFromLocal /opt/hadoop/.txt input/
hadoop jar hadoop-0.20.2-examples.jar wordcount input output
hadoop fs -cat output/

注意：mkdir -p 创建多层目录结构，
 运行文件jar的位置要着对
 -copyFromLocal 从本地拷贝， －cp 是云到云的拷贝
